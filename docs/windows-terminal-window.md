# Windows 终端窗口说明

## 为什么窗口会保留

Windows 主题启动后，需要一个长期运行的 Node.js injector 监听 Codex 的 CDP 目标。它负责首次注入、窗口变化和页面重新加载后的持续恢复，因此不是安装完成后即可退出的一次性命令。

Codex Dream Skin 上游默认隐藏该后台进程的窗口。本项目面向受管企业终端，按照仓库安全策略移除了隐藏 PowerShell、Shell 和子进程窗口的参数，所以 injector 会显示为一个持续存在的终端窗口。这是有意的安全差异，不是主题配置错误。

## 企业终端模式

这是本仓库唯一内置和测试的 Windows 模式：

1. 完全退出 Codex。
2. 双击 `安装终末地主题-Windows.cmd`，以后可双击 `启动终末地主题-Windows.cmd`。
3. 主题运行期间保留 injector 终端窗口，不要直接关闭。
4. 需要结束主题时，双击 `停用主题并恢复官方外观-Windows.cmd`。

直接关闭 injector 窗口可能不会立刻移除已经注入的 CSS，但会失去新窗口监听与重新加载后的自动恢复，主题状态会变得不可靠。

## 个人无窗口模式

个人设备可能更偏好不显示长期终端窗口，但这需要恢复隐藏子进程行为，属于不同的进程安全模型。当前仓库的企业终端规则禁止生成或分发隐藏 PowerShell、Shell、Node.js 或其他子进程的启动器，因此不会提供“个人无窗口” `.cmd`，也不会在文档中给出隐藏执行命令。

如果未来要正式支持个人模式，应作为独立发行配置进行安全评审、测试和维护，不能只在现有 `.cmd` 中追加一个隐藏参数。至少需要重新审查进程可见性、日志位置、异常退出、恢复入口和本机 CDP 端口的生命周期。

## 非等价替代方案

一次性注入可以让终端执行后退出，但它不能可靠覆盖后续创建的窗口、renderer 重载或 CDP 目标变化，因此本项目没有把它作为无窗口模式。当前持续 injector 仍是稳定性更高的方案。
